# HydrogenAtomOrbitalViewer：面向本科一年级化学教学的桌面端与 Web 端氢原子轨道交互可视化工具

作者、单位与通讯作者信息待补充

建议短标题：氢原子轨道交互可视化教学工具

## 摘要

原子轨道是普通化学、结构化学、无机化学和物理化学课程中的核心概念，但学生常难以将量子数、波函数、节面、相位和概率密度与教材中的三维轨道图像建立稳定联系。本文介绍 HydrogenAtomOrbitalViewer（HAO），一个面向本科教学开发的氢原子轨道交互可视化工具。该项目包含两个互补版本：基于 Qt/VTK 的桌面客户端，以及基于 Three.js 的纯前端 Web 版。桌面客户端适合课堂投影、离线演示和高质量图片导出；Web 版可通过浏览器直接访问，便于学生在电脑、平板或手机上进行课内外探索。两个版本均强调同一轨道的多重表征联动，包括三维等值面、电子云、轨道投影、角向函数、径向函数、径向分布函数、节面、截面和坐标轴显示。Web 版进一步加入预测-验证、轨道匹配、轨道对比、节点发现、手绘对比、讲解标注和原理演示等课堂互动模块。该工具已在山西师范大学本科一年级课堂中连续使用两年。课堂观察和学生非正式反馈表明，学生认为可旋转、可切换、可标注的多视图轨道图像有助于理解等值面与概率分布的区别、径向和角向函数的共同作用，以及节面和相位的空间含义。本文从教学需求、软件设计、数学基础、功能实现、课堂应用、活动设计、已有工具比较和后续改进等方面对 HAO 进行系统介绍。HAO 并不试图替代专业量子化学软件，而是作为一个聚焦本科教学的轨道虚拟实验室，帮助学生从被动观看静态图片转向基于证据的交互式探索。

## 1. 引言

原子轨道是化学专业学生最早接触的量子力学对象之一。在本科一年级普通化学或结构化学课程中，学生需要理解主量子数、角量子数和磁量子数，并把这些量子数与轨道大小、形状、空间取向和节点结构联系起来。在后续物理化学、无机化学和量子化学课程中，相关概念又会进一步延伸到薛定谔方程、成键模型、分子轨道理论、光谱学和结构-性质关系等内容。尽管原子轨道十分基础，但它对初学者来说并不直观。学生面对的是一个三维空间变量的数学函数，需要同时理解波函数与概率密度的区别、正负相位的意义、节面的来源，以及三维结构在二维教材插图中的投影关系。[9,11,14,15,25]

化学教育文献长期关注轨道图像的教学表达方式。早期研究通过物理模型、轮廓面、等值面和立体图来展示轨道形状；[1-5] 随后，Mathematica、Gnuplot、Winplot、Jmol 和自定义图形界面等工具被用于生成和操作轨道图像；[6-9,14] 近年来，Web 工具、点云模型、径向分布函数可视化、虚拟现实、增强现实和移动端应用又进一步拓展了学生与轨道模型互动的方式。[9,12,13,15-19] 这些工作的共同目标不是简单地“画出漂亮的轨道”，而是帮助学生理解图像背后的量子数、径向函数、角向函数、节面、相位和概率密度。

若干近年工作对本项目具有直接启发。Hanson 报道了面向本科物理化学的氢原子轨道 GUI 工具，强调高质量三维体密度图和节面显示在轨道教学中的价值。[14] Li 等开发的 Online Orbital Explorer 和 BingOrbital Game 将原子轨道可视化与探究式学习、数据收集和检索练习结合起来，体现了从“展示图像”到“支持学习过程”的转变。[12,13] Manae 和 Hazra 指出，学生容易把轨道形状错误地归因于角向函数，而忽略径向函数与角向函数的共同作用。[11] Tully 等通过点描式 Web 可视化强调电子云边界的弥散性和概率本质。[9] Rachek 和 Wilson 则集中讨论径向分布函数，展示动态 RDF 图像如何帮助学生理解电子在不同核距处出现的概率。[15]

HydrogenAtomOrbitalViewer（HAO）正是在上述教学需求和文献背景下开发的。项目的核心思想是：学生不应只看到一张轨道边界面图，而应在同一工作空间中同时观察完整轨道、角向函数、径向函数或径向分布函数，以及轨道投影。这样，教材中分散出现的多个图像不再是彼此独立的静态插图，而是同一数学对象的不同可操作视图。HAO 的桌面客户端采用 Qt 和 VTK 构建，适合教师离线演示、课堂投影和图片导出；Web 版采用浏览器端计算和 Three.js 渲染，适合学生通过链接直接访问。

本文介绍 HAO 的设计思想、数学基础、桌面端与 Web 端实现、课堂使用方式和探究活动设计。该工具已在山西师范大学本科一年级课堂连续使用两年，学生反馈显示其对理解抽象轨道概念具有积极帮助。本文的贡献主要是教学和技术整合，而不是提出新的量子化学计算方法。HAO 的定位是一个面向本科基础教学的轨道虚拟实验室。

## 2. 教学设计目标

HAO 的开发围绕五个教学设计目标展开。

第一，降低学生使用门槛。许多专业量子化学和分子可视化软件能够显示轨道，但通常需要计算输入文件、输出文件、后处理流程或较高的专业背景。HAO 允许学生通过量子数或熟悉的轨道名称直接选择目标轨道，并立即生成图像。

第二，强调多重表征并列呈现。单独的边界面图虽然直观，但容易隐藏概率分布、径向节点和内部结构。[9,14] HAO 将等值面、电子云、径向分布、角向函数和投影图放在同一工作流程中，使学生能比较不同表征各自回答的问题。[9,11,15,16,25]

第三，显式呈现节点和相位。许多学生能记住“总节点数等于 n-1”等规则，却难以在三维图像中指出节点的位置。[12-14] HAO 提供径向、极向和经向节点显示，配合截面、裁切和正负相位颜色，帮助学生把符号规则转化为空间判断。

第四，支持课堂互动，而不仅是教师演示。Web 版中的预测-验证、轨道匹配、轨道对比、节点发现、讲解标注和手绘对比等模块，使学生能够先提出预测，再用软件生成图像进行验证和修正。[12,13]

第五，适应真实教学环境。桌面端可离线运行并导出高质量图片，适合教师制作课件、讲义和论文插图；Web 端可作为静态网页部署，学生只需打开链接即可使用，不需要安装 Qt、VTK 或其他依赖。

### 目标学生与学习目标

HAO 的主要对象是普通化学、结构化学或基础量子化学课程中的本科一年级学生。若在物理化学课程中使用，也可作为氢原子薛定谔方程解的图形化补充。使用前，学生应初步了解量子数、轨道名称和概率密度的基本含义。

完成基于 HAO 的学习活动后，学生应能够：

- 根据 `n`、`l`、`m` 或轨道名称预测轨道的大致形状、取向、相位和节点数；
- 区分波函数相位与概率密度；
- 在三维轨道图中识别径向节点和角向节点；
- 解释为什么角向类型相同的轨道在主量子数不同时仍会呈现不同径向结构；
- 比较等值面、电子云、径向函数、角向函数、轨道投影和截面图各自表达的信息；
- 用图像证据说明轨道判断依据，而不是只背诵固定图形。

## 3. 数学与计算基础

HAO 显示的是类氢原子轨道，其波函数可写成径向部分和角向部分的乘积；这一处理也与已有轨道可视化文献中强调径向和角向共同作用的思路一致：[11,14,15]

```text
psi(n,l,m; r, theta, phi) = R(n,l; r) Y(l,m; theta, phi)
```

其中 `n` 为主量子数，`l` 为角量子数，`m` 为磁量子数。三者满足：

```text
n = 1, 2, 3, ...
l = 0, 1, ..., n - 1
m = -l, ..., 0, ..., l
```

为了匹配本科化学中常见的轨道图像，程序采用实轨道组合来表示 `Px`、`Py`、`Pz`、`Dxy`、`Dxz`、`Dyz`、`Dx2-y2` 和 `Dz2` 等轨道名称。[14] 正相和负相使用不同颜色显示。这样的处理方式符合基础化学课程的表述习惯，能够帮助学生理解轨道相位、节点和后续成键概念，同时避免在入门阶段过早引入复球谐函数的完整相位表示；更高阶的复值轨道相位可视化已有专门文献讨论。[24]

HAO 中不同窗口对应同一波函数的不同信息。

原子轨道窗口显示轨道的三维空间形态。桌面端使用 VTK 渲染等值面、网格、光滑面、电子云、坐标轴、盒子、截面和节面；Web 端使用 Three.js 和 Marching Cubes 在浏览器中生成正负相位等值面。

电子云模式用空间点分布表达概率密度。该模式适合解释“轨道不是硬边界”，能够避免学生把等值面误解为电子运动的固定外壳。[9]

径向窗口显示 `R(r)`、`R(r)^2` 或径向分布函数。`R(r)` 适合讨论符号变化和径向节点，径向分布函数更适合讨论电子在不同核距附近出现的概率。[11,15]

角向函数窗口突出轨道方向性。它有助于解释 p、d、f 等轨道的角向取向，但必须与完整轨道和径向图共同使用，以避免学生误以为轨道形状只由角向函数决定。[11]

轨道投影窗口把三维波函数信息转换为二维曲面和等高线式图像。该窗口适合连接三维瓣、正负相位、投影平面和节点结构。

节面显示基于类氢原子轨道的节点关系：

```text
角向节点数 = l
径向节点数 = n - l - 1
总节点数 = n - 1
```

这些公式在教材中非常简洁，但学生往往难以把它们转化为三维空间图像。HAO 的作用正是让这些规则变成可观察、可旋转、可比较的图形证据。

## 4. 桌面客户端

HAO 的原始桌面客户端是一个 Qt/VTK 程序。界面左侧为控制面板，右侧为多窗口可视化区域。氢原子轨道模块通常包含四个联动窗口：轨道投影、原子轨道、径向分布函数和角向函数。用户可以通过量子数或轨道名称选择目标轨道，并调节正瓣、负瓣、网格、光滑、电子云、投影、配色、截面、节面、盒子尺寸、格点数、间隔、等值面和透明度等参数。

桌面端尤其适合教师课堂演示。教师可以在同一屏幕中展示完整轨道、径向函数、角向函数和投影图，让学生直接比较它们之间的关系。例如，比较 `2Pz` 和 `3Pz` 时，学生能够看到二者角向形状相似，但 `3Pz` 由于径向函数多出一个节点而呈现不同的内部结构。教师还可以导出 PNG 图片，用于课件、讲义、学生报告和论文插图。

从技术上看，Qt 负责菜单、工具栏、选项卡、参数面板、对话框和语言切换；VTK 负责三维科学可视化，包括等值面、网格、光滑面、点云、坐标轴、盒子、截面和节面等对象。这一架构具有较好的课堂稳定性和图像质量。

## 5. Web 版

Web 版的开发目的是让学生不依赖本地 Windows 环境也能使用 HAO。该版本是纯前端静态应用，基于 Vite、Three.js 和浏览器端 JavaScript 实现。部署后，用户只需使用支持 WebGL 的现代浏览器打开网页链接即可运行，不需要安装 Qt、VTK、Node.js 或 Python。

Web 版保留了桌面客户端的核心四窗口布局：轨道投影、原子轨道、径向分布函数和角向函数。同时，它尽量复现桌面端的控制逻辑，包括量子数、轨道名称、正负相位颜色、网格/光滑显示、电子云、坐标轴、截面、盒子半径、格点数、等值面、透明度和图片导出等。Web 端使用 Three.js 进行三维渲染，并利用 Marching Cubes 在浏览器中生成轨道等值面。

在复现桌面端视觉效果的基础上，Web 版进一步加入了面向课堂互动的模块：

- 预测-验证模式：学生先预测轨道形状、相位和节面，再显示计算结果进行比较；
- 预测评分：从节点数、相位、取向和投影理解等方面给出结构化反馈；
- 错误诊断：引导学生识别“把相位当概率密度”“忽略径向节点”等常见错误；
- 节点发现：通过显示和隐藏节点帮助学生建立量子数与节点的联系；
- 轨道对比：把两个轨道并列显示，比较径向节点、角向特征和空间大小；
- 轨道配对挑战：以检索练习方式训练学生根据图像证据识别轨道；
- 原理演示：以类似视频的分步讲解方式展示当前轨道如何由波函数、径向部分、角向部分、等值面、投影和 RDF 生成；
- 讲解标注：教师可在图像上添加“正相”“负相”“节点”“投影平面”“径向节点”等可拖动标签；
- 手绘对比：学生可先绘制自己的预测图，再与软件生成结果对比。

这些功能受到 JCE 近期关于探究式活动和游戏化轨道学习工作的启发，[12,13] 使 Web 版不只是桌面端的在线替代品，而是一个更适合课堂活动和学生自主学习的平台。

当前 Web 版部署地址为：

```text
https://zhangfq-chemistry.github.io/HAO-Web/
```

由于该应用在浏览器端完成计算和渲染，普通使用不需要后端服务器和数据库。教师可以将其部署到 GitHub Pages、学校服务器或其他静态网页托管平台。

## 6. 课堂实施

HAO 已在山西师范大学本科一年级课堂中连续使用两年。课堂实施主要围绕普通化学或结构化学中的原子结构内容展开，重点包括量子数、轨道形状、相位、概率密度、径向节点、角向节点和轨道投影。桌面端主要用于教师课堂演示和高质量投影，Web 端则用于学生课内外通过链接访问和自主探索。

课堂活动通常采用“预测-观察-解释”的流程。教师先给出轨道名称或量子数，要求学生预测轨道形状、正负相位、节点数和可能的截面特征。随后，教师或学生使用 HAO 生成轨道图像，对比预测和结果。最后，学生需要解释哪些特征来自径向函数，哪些来自角向函数，不同图像表征分别支持什么结论。

该工具也被用于轨道识别和检索练习。教师可以展示去掉名称的轨道图片，要求学生根据瓣数、相位、节面、径向结构和投影特征判断轨道名称，并说明判断依据。与单纯记忆轨道形状相比，这种活动更强调学生从图像中提取证据并进行推理。[12,13,25]

两年的课堂使用反馈总体积极。学生的非正式反馈表明，可旋转的三维图像、电子云与等值面的切换、截面显示、径向图和角向图并列呈现，有助于他们把抽象的轨道概念转化为可观察的空间结构。教师观察到，当软件中可以直接显示节点、相位和投影时，学生更愿意围绕图像证据讨论“为什么是这个轨道”。需要说明的是，目前这些反馈尚未通过严格的前后测或对照实验收集，因此更适合作为初步课堂实施证据，而不是正式的学习效果统计结论。

若后续将本文发展为包含教学评价数据的研究论文，可进一步加入前后测、图像解释题评分、学生手绘轨道对比、问卷调查、访谈和延迟测验等证据。

| 轨道匹配 | 训练图像识别和证据推理 | 使用 Web 端轨道配对挑战 | 团队答案和判断依据 |
| 原理演示 | 连接公式、函数和渲染结果 | 使用 Web 端分步讲解模式 | 从波函数到图像的概念图 |

这些活动可按照探究式学习的基本流程组织：情境导入、提出预测或问题、用软件收集证据、形成结论、同伴交流和迁移应用。[12,13] 教师应把提问重点放在证据上，例如：“节点在哪里？”“哪一个窗口能证明你的判断？”“颜色表示相位还是概率？”“当 n 变大而 l 不变时，图像为什么改变？”

## 8. 与已有工具的比较

HAO 在已有轨道可视化工具中具有明确定位。它不如通用量子化学软件功能全面，[20-23] 也不如 VR/AR 工具具有沉浸感，[17-19] 但它更聚焦于本科基础轨道教学，更容易部署在普通课堂中，也比静态教材图片和脚本绘图更适合交互探索。[6-8]

### 表 2. HAO 与相关工作的关系

| 方向 | 代表性工作 | 与 HAO 的关系 |
| --- | --- | --- |
| 早期轨道模型和轮廓面 | Lambert；Ogryzlo 和 Porter；David | HAO 继承其可视化轨道形态的目标，但采用交互式桌面和 Web 渲染 |
| 脚本或桌面绘图工具 | Ramachandran 和 Kong；Moore；Chung | HAO 降低脚本绘图门槛，学生可直接通过界面选择轨道和显示模式 |
| 点云和 Web 轨道可视化 | Tully 等 | HAO 同样强调电子云和概率分布，并将其与径向、角向、投影和等值面联动 |
| 径向-角向共同作用 | Manae 和 Hazra | HAO 将径向和角向窗口放在完整轨道旁，直接回应“轨道形状不能只看角向函数”的问题 |
| 探究式活动和轨道游戏 | Li 等 | Web 版加入预测-验证、轨道配对、轨道对比和课堂互动模块 |
| 氢原子轨道 GUI | Hanson | HAO 同样服务氢原子轨道可视化，但增加 Qt/VTK 桌面端、Web 端和课堂活动功能 |
| 径向分布函数可视化 | Rachek 和 Wilson | HAO 将 RDF 放入三维轨道、角向函数和投影同一工作流中 |
| 通用分子可视化软件 | VMD、Avogadro、Gabedit、Multiwfn | HAO 科学功能较窄，但对本科轨道教学更直接 |
| VR、AR 和移动端工具 | Dai 等；Levy 等；Zambri 和 De Backere | HAO 不追求沉浸式体验，但更容易通过课堂电脑和网页链接推广 |

因此，HAO 的主要贡献不是计算方法创新，而是教学整合创新：它把多种轨道表征、桌面端高质量渲染、Web 端易访问性和课堂互动任务整合到一个连续的教学工作流中。

## 10. 安全说明

HAO 为计算机软件教学活动，不涉及化学品、生物样品、放射性材料或实验室危险。使用时只需遵守普通计算机和投影设备操作规范。

## 11. 软件可获得性与可复现性

Web 版可作为静态前端网页发布，学生可通过浏览器访问。桌面端可在安装 Qt 和 VTK 后从源代码构建，适合 Windows 环境下的课堂演示和图片导出。为保证文章插图可复现，建议记录轨道名称、量子数、等值面、格点数、盒子半径、透明度、显示模式、相位颜色、截面设置和相机视角等参数。

建议软件可获得性表述如下：

HAO Web 版可通过 `https://zhangfq-chemistry.github.io/HAO-Web/` 访问。项目源代码、构建说明和使用文档可由项目仓库获得。桌面客户端依赖 Qt 和 VTK，主要用于 Windows 课堂演示、本地图像生成和离线教学。

## 12. 局限性与后续工作

当前版本仍存在若干局限。首先，HAO 是教学可视化工具，不是通用量子化学软件，不能替代电子结构计算、波函数分析或分子轨道后处理程序。其次，当前版本主要面向实轨道和教学相位显示，尚未提供完整复波函数相位色轮。第三，电子云显示主要用于定性解释概率分布；若用于定量比较，需要固定采样方法、点数、随机种子和显示参数。第四，Web 端性能受到浏览器和显卡影响，较高格点数和复杂高阶轨道在低性能设备上可能不够流畅。第五，虽然该工具已在课堂使用两年并获得积极反馈，但尚未完成严格的学习效果评价研究。

后续工作可从四个方向展开。第一，开展系统教学评价，包括前后测、学生手绘图评分、延迟测验和访谈。第二，优化 Web 端性能，例如使用 Web Worker 或 GPU 加速提高高格点渲染流畅度。第三，开发活动编辑功能，使教师能够保存自定义任务、提示语、评分标准和答案。第四，完善移动端、双语界面和可访问性支持，使平板课堂和跨设备使用更加稳定。

## 13. 结论

HydrogenAtomOrbitalViewer 为氢原子轨道教学提供了一个聚焦、可交互、可部署的多视图可视化环境。桌面客户端使用 Qt 和 VTK 支持课堂投影、多窗口联动和高质量图片导出；Web 版使用 Three.js 和浏览器端计算，使学生可以通过静态链接直接访问，并加入预测、匹配、对比、标注、手绘和原理演示等课堂互动功能。该工具已在山西师范大学本科一年级课堂连续使用两年，用于支持量子数、相位、概率密度、径向节点、角向节点和轨道投影等内容的教学。HAO 最重要的教学价值在于多重表征整合：学生可以在同一工作流程中比较边界面、电子云、径向图、角向图、投影图和截面图各自揭示的轨道信息。通过这种方式，原子轨道学习可以从被动观看静态教材图像转变为基于证据的交互式探索。
